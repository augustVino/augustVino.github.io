首先，先介绍一下 CSS3 新增的 animation 属性。

animation: ani 5s linear 0s infinite alternate;

@keyframes ani1{
    0%{}

    100%{}

    from{}
    to{}
}

copy success
Copy successed

  • @kayframes 定义关键帧动画

  • animation-name 动画名称

  • animation-duration 动画时间(单位是s或者ms)

  • animation-timing-function 动画曲线

    • linear(匀速)，等同于贝塞尔曲线(0.0, 0.0, 1.0, 1.0)；
    • ease(逐渐变慢)，等同于贝塞尔曲线(0.25, 0.1, 0.25, 1.0)；
    • step(步数)
    • ease-in(加速)，等同于贝塞尔曲线(0.42, 0, 1.0, 1.0)；
    • ease-out(减速)，等同于贝塞尔曲线(0, 0, 0.58, 1.0)；
    • ease-in-out(先加速后减速)，等同于贝塞尔曲线(0.42, 0, 0.58, 1.0)；
    • cubic-bezier(自定义贝塞尔曲线)自定义贝塞尔曲线 cubic-bezier(<number>, <number>, <number>, <number>)

  • animation-delay 动画延迟(单位是s或者ms)

  • animation-iteration-count 动画播放次数 n | infinite

  • animation-direction 动画结束后是否反向还原 normal | alternate

  • animation-play-state 动画状态 paused(停止) | running(运动)

  • animation-fill-mode 动画前后的状态 none | forward(结束时停留在最后一帧) | backwards(开始时停留在定义的开始帧) | both(前后都应用)

# 附一：给定gif图，如何生成一帧一帧的png

如果UI直接给的是gif图（有些UI或技术有限或为了省事儿），前端肯定不能直接拿来用（gif会引起页面周期性的重绘，使用GIF的带宽流量是Video(h264)格式的12倍，而播放性能消耗是2倍。性能非常的差），工作以来通常有两种处理方式：

  1. 把gif图转换为webp格式，转换方式看这里
  2. 把gif转成雪碧图，然后通过css3 @keyframs实现动画效果。 此处详细介绍方案二。

# 一、gif图抽帧

# 二、生成雪碧图

上步已经在指定的文件夹中生成了给定gif的帧图片（png）。通过css3 @keyframes绘制动画时，肯定不能直接用这些图片。因为gif生成的帧图片数量肯定不少，如果直接用的话，会消耗多少个http请求。。所以这里先把这些图片合并成雪碧图的形式：看这里.可以根据实际情况和个人喜好来调整生成雪碧图的排布规则。这里我选择了0间距、二叉树的方式排列（此外还有由左到右、由上到下排列）。

生成雪碧图后，左侧还有对应的位置代码，可以直接拿来用，省的计算了。。

# 附二：如果页面中帧动画很多，如何快速生成代码？

由于需求中帧动画有很多，每次计算雪碧图的位置很费事儿，故写了个less小循环，用来快速生成@keyframes代码。

// style.less

// 92代表的是帧动画的帧数-1
.loop(@counter) when (@counter<=92) {
    @percent: round(@counter*100/92%, 3);

    @{percent} {
        transform: translate3d(-@counter*120px, 0, 0);
    }

    .loop(@counter+1);
}

@keyframes name {
    .loop(0);
}

Copy successed

注意：此循环代码只适用于雪碧图是横向排列的，如有其它排列规则的需要相应的修改此处代码
